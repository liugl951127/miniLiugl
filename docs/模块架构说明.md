# 模块架构说明 (V3.5.5+)

## 16 模块分类

### 1 个 Library 公共模块
| 模块 | packaging | main class | 用途 |
|------|-----------|------------|------|
| `minimax-common` | **pom** | 无 | 公共工具/异常/限流/缓存/异步 |

⚠️  **不要**作为 Docker 服务运行, **不要**打成 jar. 仅供其他模块通过 maven 依赖.

## packaging=pom 的影响

### minimax-common (pom) vs 其他 (jar)

```
minimax-common/
├── pom.xml         ← <packaging>pom</packaging>
├── src/main/java/  ← 源码存在
└── target/
    ├── classes/    ← 编译后 class 文件 (供依赖用)
    └── (无 jar)    ← 不打 jar

minimax-ai/
├── pom.xml
├── src/main/java/
└── target/
    ├── classes/
    ├── minimax-ai.jar              ← 普通 jar (200KB)
    └── minimax-ai-spring-boot.jar  ← Spring Boot fat jar (110MB)
```

### mvn 命令差异

| 操作 | minimax-common | 其他模块 |
|------|---------------|----------|
| `mvn install` | ✓ 装到本地 repo | ✓ 装到本地 repo |
| `mvn package` | ✗ (无 jar) | ✓ 打 spring-boot fat jar |
| `mvn -am compile` | ✓ (被依赖) | ✓ |
| Docker build | ✗ (跳过) | ✓ (生成镜像) |
| Docker run | ✗ (不需要) | ✓ (16 个服务) |

## 验证工具

`bash scripts/check-services.sh` 自动检查:
1. 所有 pom.xml 的 packaging 类型
2. 所有 @SpringBootApplication 主类存在
3. docker-compose.yml / docker-compose.mini.yml 不包含 common
4. @TableField 命名规范

## 添加新模块

### 1. 添加可执行应用模块 (16 个中再 +1)
```bash
mkdir -p backend/minimax-newservice
# 1. 写 application.yml (含 server.port)
# 2. 写 *Application.java (含 @SpringBootApplication)
# 3. pom.xml: <packaging>jar</packaging>
# 4. mvn install
# 5. 添加到 docker-compose.yml
# 6. 跑 bash scripts/gen-nginx-config.sh
```

### 2. 添加 library 公共模块
```bash
mkdir -p backend/minimax-newlib
# 1. 写 pom.xml: <packaging>pom</packaging>  ⚠️
# 2. 不要 @SpringBootApplication
# 3. 不要 spring-boot-maven-plugin
# 4. mvn install (装到本地 repo)
# 5. 其他模块添加依赖
```
