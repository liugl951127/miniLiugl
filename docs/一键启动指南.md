# MiniMax Platform 一键启动指南 (V3.5.5+)

## TL;DR

```bash
cd /path/to/miniLiugl
./start.sh           # 智能选择模式
```

智能选择逻辑:

| 环境 | 内存 | 自动选择 |
|------|------|----------|
| 无 Docker / < 1.2GB | < 1.2GB | H2 沙箱 (单 java 进程, 512MB) |
| 有 Docker / 1.2-4GB | 1.2-4GB | Docker 精简 6 容器 (1.5GB) |
| 有 Docker / 4GB+ | 4GB+ | Docker 完整 21 容器 (6GB) |

## 三种模式

### 1. H2 沙箱 (推荐开发/沙箱)

```bash
./start.sh h2
```

**特点**:
- 单 java 进程 (`minimax-ai`)
- 内存数据库 H2, 89 表 + 59 INSERT 自动导入
- 端口 8094
- 内存 ~512MB
- **任何环境都能跑** (只要有 Java 17+)

**适用**:
- 沙箱 / 树莓派 / 2GB VPS
- 快速验证 AI 功能
- 单元测试 / 集成测试

**功能**: 用户登录 + AI 核心 7 大模块 (知识库/意图/智能体群/Raft/推送/License/Spark)

### 2. Docker 精简 6 容器 (推荐小内存 VPS)

```bash
./start.sh mini
```

**6 容器**:
```
mariadb + redis + auth + ai + gateway + nginx
```

**特点**:
- 单端口 80 (nginx 反代)
- MySQL/MariaDB + Redis 持久化
- 端口 80/3306/6379/7080/8081/8094
- 内存 ~1.5GB
- **生产可用的精简版**

### 3. Docker 完整 21 容器 (推荐 8GB+ VPS)

```bash
./start.sh full
```

**21 容器**:
- 3 基础设施 (mariadb + redis + nacos)
- 17 微服务 (含 minimax-ai)
- 1 nginx 反代

**特点**:
- 完整业务能力 (chat/memory/rag/agent/admin 等)
- 内存 ~6GB
- 启动 ~3 分钟

## 管理命令

## 安装依赖

### 仅需 Java (H2 沙箱)

```bash
# Debian/Ubuntu
sudo apt-get install -y openjdk-17-jdk

# CentOS/RHEL
sudo yum install -y java-17-openjdk
```

### 需要 Docker (mini/full)

```bash
# 一键安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装 docker compose
sudo apt-get install -y docker-compose-plugin   # Debian/Ubuntu
sudo yum install -y docker-compose-plugin       # CentOS/RHEL
```

# {"status":"UP",...}

## 故障排查

### H2 沙箱启动失败

```bash
# 1. 看日志
tail -f /tmp/loa/ai.log

# 2. 常见: 端口占用
lsof -i :8094
# 解决: ./start.sh stop

# 3. 常见: 内存不足
free -h
# 解决: 加内存 / 用 Docker 精简
```

### Docker 启动失败

```bash
# 1. Docker daemon
sudo systemctl start docker
sudo systemctl status docker

# 2. 看日志
./start.sh logs

# 3. 重新构建
./start.sh stop
./start.sh mini   # 重新 build
```
