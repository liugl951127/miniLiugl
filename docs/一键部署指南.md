## 🔧 部署模式详解

### 1. 智能自动模式 (推荐)

```bash
sudo ./deploy-all.sh up
```

根据系统内存自动选择:
- **< 2GB** → H2 沙箱 (单 jar, 512MB)
- **2-4GB** → Docker 精简 (6 容器, 1.5GB) ← 沙箱默认
- **4GB+** → Docker 完整 (21 容器, 8GB+)

### 2. 强制指定模式

```bash
sudo ./deploy-all.sh up mini    # Docker 精简 6 容器
sudo ./deploy-all.sh up full    # Docker 完整 21 容器
sudo ./deploy-all.sh up host    # 宿主机 systemd/nohup
sudo ./deploy-all.sh up h2      # H2 沙箱单服务
```

### 3. 精简模式 (deploy-mini.sh)

6 个容器: `mariadb + redis + auth + ai + gateway + nginx`

```bash
./deploy-mini.sh up        # 一键启动 (自动构建前端 + 后端)
./deploy-mini.sh rebuild   # 完整重建 (前端 + 后端 + 镜像)
./deploy-mini.sh status    # 查看容器/资源/端口
./deploy-mini.sh logs      # 实时日志
./deploy-mini.sh restart   # 重启
./deploy-mini.sh down      # 停止 (保留数据)
./deploy-mini.sh clean     # 清理 (危险)
```

**deploy-mini.sh up 实际做的事**:
1. 检查 Node.js (18+), 用 Vite 构建前端 → `frontend/dist/`
2. 检查 Maven, 编译后端核心 3 模块 → `target/*.jar`
3. 启动 `docker-compose.mini.yml` (6 容器)
4. nginx 容器挂载 `frontend/dist` + 反代 auth/ai/gateway
5. 等待健康检查通过, 输出访问入口

## 🏗️ 架构总览

```
┌────────────────────────────────────────────────────────┐
│  Browser (单端口 80, 无 CORS, nginx 代理)              │
│      ↓                                                  │
│  ┌─────────────────────────────────────────┐           │
│  │  nginx 容器 (端口 80)                   │           │
│  │  - /           → frontend/dist SPA      │           │
│  │  - /healthz    → ok                     │           │
│  │  - /api/v1/auth/*  → auth:8081          │           │
│  │  - /api/v1/ai/*    → ai:8094            │           │
│  │  - /api/v1/*       → gateway:7080       │           │
│  └────┬──────────┬──────────┬──────────────┘           │
│       ↓          ↓          ↓                          │
│    [auth]      [ai]      [gateway]                     │
│   (8081)      (8094)     (7080)                        │
│       ↓          ↓          ↓                          │
│   ┌──────────────────────────────┐                     │
│   │  mariadb  ←→  redis          │                     │
│   │  (3306)      (6379)           │                     │
│   └──────────────────────────────┘                     │
└────────────────────────────────────────────────────────┘
```

**优势**:
- 单端口 80 入口, 防火墙只开 1 个端口
- 前端和 API 同源, 无 CORS
- nginx 代理, 性能 + 安全
- vue-router history 模式, URL 干净
- H5 移动端可访问 (Capacitor 包装可选)

## 🔄 升级与维护

### 升级代码后重启

```bash
# 拉取最新代码
git pull origin main

# 重新部署 (自动重建前端 + 后端)
./deploy-mini.sh up
# 或仅重启
./deploy-mini.sh restart
```

### 升级镜像版本

```bash
./deploy-mini.sh rebuild   # 完整重建 (慢, 5-10 分钟)
./deploy-mini.sh up        # 启动新镜像
```

### 数据库迁移

```bash
# 增量修复 (新增表/字段)
mysql -uroot -proot123456 minimax_platform < sql/complete.sql

# 重置 (清空重建)
./deploy-mini.sh clean
./deploy-mini.sh up
```

## 🐛 故障排查

| 现象 | 排查 |
|------|------|
| 80 端口冲突 | `sudo lsof -i :80` 杀掉占用进程 |
| 启动后 502 Bad Gateway | `docker compose logs gateway` 看后端是否就绪 |
| 前端空白 | 检查 `frontend/dist/index.html` 存在, nginx 容器挂载成功 |
| 数据库连接失败 | 等 mariadb 30s 完成 init (首次启动会跑 complete.sql) |
| 内存不足 OOM | 用 `./deploy-all.sh up mini` 精简模式 |

```bash
# 看容器日志
./deploy-mini.sh logs              # 全部
docker logs minimax-nginx          # 单个容器

# 进容器调试
docker exec -it minimax-ai sh

# 重新跑 SQL
docker exec -i minimax-mariadb mysql -uroot -proot123456 minimax_platform < sql/complete.sql
```

## 🌐 公网部署 (VPS / 服务器)

### CentOS 9 / RHEL

```bash
# 1. 安装 Docker
sudo dnf install -y docker docker-compose-plugin
sudo systemctl enable --now docker

# 2. 防火墙开 80
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload

# 3. 部署
sudo ./deploy-all.sh up

# 4. 访问
http://<your-server-ip>/
```

### Ubuntu 20+ / Debian 11+

```bash
sudo apt update && sudo apt install -y docker.io docker-compose-plugin
sudo systemctl enable --now docker
sudo ./deploy-all.sh up
```

### 域名 + HTTPS (推荐)

```bash
# 1. DNS A 记录: your-domain.com → <server-ip>
# 2. 安装 certbot
sudo apt install -y certbot python3-certbot-nginx

# 3. 申请证书
sudo certbot --nginx -d your-domain.com

# 自动续期 (cron)
echo "0 3 * * * certbot renew --quiet" | sudo crontab -
```
