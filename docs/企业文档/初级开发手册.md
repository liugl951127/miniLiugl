## 第 1 章 项目介绍

### 1.1 MiniMax Platform 是什么?

MiniMax Platform 是一个**自研多模态 AI 平台**, 集成了:
- 17 个微服务 (16 业务 + 1 自研 AI)
- 21 个 AI 工具
- Vue 3 前端 (支持 PWA + Capacitor 移动端)
- 自研大模型 (MiniTransformer)
- 智能体群组 (4 协作策略)
- 模型市场 (版本+License+计费)
- 集群联邦 (多节点+选举+路由)
- 移动推送 (Web/APNs/FCM)
- 训练可视化 (实时曲线+Checkpoint)
- 数据看板 (11 指标+SSE)

### 1.2 技术栈

**后端**:
- Java 17 + Spring Boot 3.x
- Spring Cloud (Gateway, OpenFeign)
- MyBatis-Plus 3.x
- MySQL 8.0 / H2 (测试)
- Redis 7
- Nacos (配置中心)
- WebSocket / STOMP
- Apache POI / ONNX Runtime / CRDT

**前端**:
- Vue 3 + Composition API
- TypeScript
- Element Plus
- Pinia (状态管理)
- Vue Router
- Vite (构建)
- Vitest (测试)
- ECharts
- Axios
- PWA (Service Worker)
- Capacitor (移动端)

**基础设施**:
- Docker / Docker Compose
- Nginx (反向代理)
- OpenTelemetry Collector
- Prometheus / Grafana (可选)

```
minimax-common (基础)
  ↓
所有业务模块
  ↓
```

`minimax-ai` 独立, 不依赖其他业务模块, 可单独部署.

### 3.4 数据流向

```
请求 → Gateway → 鉴权 → 路由 → 业务模块 → DB/Redis
                                          ↓
响应 ← JSON ← Biz Logic ← Service ← Mapper
```

- **MyBatis-Plus**: ORM, 简化 CRUD
- **Nacos**: 动态配置, 服务发现 (可选)
- **Spring Security**: 鉴权, BCrypt 密码
- **JWT**: 无状态 token
- **WebSocket / SSE**: 实时推送
- **OpenFeign**: 服务间调用

## 第 4 章 代码规范

### 4.1 Java 规范

**命名**:
- 类名: `PascalCase` (UserService)
- 方法/变量: `camelCase` (getUserById)
- 常量: `UPPER_SNAKE` (MAX_RETRY)
- 包名: 全小写 (`com.minimax.ai`)

**注释**:
- 类: JavaDoc 完整说明
- 方法: 关键参数/返回值
- 关键逻辑: 行内注释

**Lombok**:
```java
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class User {
    private Long id;
    private String name;
}
```

### 4.2 前端规范

**Vue 3 + Composition API**:
```vue
<script setup>
import { ref, computed, onMounted } from 'vue'
const count = ref(0)
const double = computed(() => count.value * 2)
onMounted(() => console.log('mounted'))
</script>
```

**ESLint + Prettier** (项目已配, 保存自动格式化).

**命名**:
- 组件: `PascalCase.vue` (UserCard.vue)
- 工具: `camelCase.js` (formatDate.js)
- 常量: `UPPER_SNAKE.js`

### 4.3 Git 规范

**分支**:
- `main`: 主分支, 受保护
- `feature/xxx`: 功能分支
- `fix/xxx`: 修复分支
- `release/xxx`: 发布分支

**Commit**:
- feat: 新功能
- fix: 修复
- docs: 文档
- style: 格式
- refactor: 重构
- test: 测试
- chore: 杂项

**示例**:

## 第 5 章 开发流程

### 5.1 完整流程

```
1. 拉主分支最新代码
   git checkout main
   git pull origin main

2. 创建功能分支
   git checkout -b feature/my-feature

3. 编码 (IDE 自动保存)

4. 写测试 (TDD: 先写测试再写实现)

5. 跑测试
   mvn test -pl minimax-xxx
   npm run test:unit

6. 自测
   - 单元测试全过
   - 手动跑功能
   - 边界场景

7. 提交
   git add .
   git commit -m "feat: my feature"
   git push origin feature/my-feature

8. 创建 PR
   - 描述改动
   - 关联 issue
   - 等 review

9. 合入
   管理员 review 通过 → 合并到 main

10. 删除分支
    git branch -d feature/my-feature
```

### 5.2 第一个任务示例

**任务**: 添加 "echo" 工具, 调用返回 "Hello, {name}!"

**步骤**:

1. 创建分支
   ```bash
   git checkout -b feature/echo-tool
   ```

2. 后端: 添加工具类
   ```java
   // backend/minimax-ai/src/main/java/com/minimax/ai/tool/EchoTool.java
   @Component
   public class EchoTool extends AbstractSimpleTool {
       @Override
       public String code() { return "echo"; }
       @Override
       public String execute(AiTool tool, Map input) {
           return "Hello, " + input.get("name") + "!";
       }
   }
   ```

3. 写测试
   ```java
   @Test
   void testEcho() {
       EchoTool t = new EchoTool();
       assertEquals("Hello, World!", t.execute(null, Map.of("name", "World")));
   }
   ```

4. 跑测试
   ```bash
   cd backend
   mvn -pl minimax-ai test -Dtest=EchoToolTest
   ```

5. 前端: 添加 API
   ```javascript
   // frontend/src/api/tools.js
   export const echo = (name) => http.post('/ai/tools/echo', { name })
   ```

6. 前端: 添加 UI
   ```vue
   <template>
     <div>
       <el-input v-model="name" />
       <el-button @click="onClick">Echo</el-button>
       <p>{{ result }}</p>
     </div>
   </template>
   ```

7. 跑前端测试
   ```bash
   npm run test:unit
   ```

8. 提交 + 推 PR
   ```bash
   git add .
   git commit -m "feat(ai): add echo tool"
   git push origin feature/echo-tool
   ```

## 第 8 章 常见错误

### 8.1 后端常见错误

**E1: `Could not find artifact com.minimax:minimax-common`**
- 原因: 父模块未编译
- 解决: `mvn install -DskipTests` (在 backend 根目录)

**E2: `Communications link failure`**
- 原因: MySQL 未启动或连接失败
- 解决: 启动 MySQL, 检查 application.yml 配置

**E3: `JWT secret not configured`**
- 原因: JWT 密钥未设置
- 解决: 设置环境变量 `JWT_SECRET=xxx` 或 application.yml

**E4: `端口被占用`**
- 原因: 8080/7080 等端口被占
- 解决:
  ```bash
  lsof -i :7080
  kill -9 <pid>
  ```

**E5: `OutOfMemoryError`**
- 解决: 加 JVM 参数 `-Xmx2g`

### 8.2 前端常见错误

**E6: `Cannot find module '@/api/...'`**
- 原因: @ 别名未配
- 解决: 检查 `vite.config.js` 的 `resolve.alias`

**E7: `Failed to fetch`**
- 原因: 后端未启动 或 CORS
- 解决: 启动后端, 检查 `vite.config.js` 的 proxy

**E8: `npm install` 卡住**
- 解决: 换镜像
  ```bash
  npm config set registry https://registry.npmmirror.com
  ```

**E9: 浏览器报错 `SSE error`**
- 原因: 反向代理未配置
- 解决: nginx 加 `proxy_buffering off`

### 8.3 Git 常见错误

**E10: `merge conflict`**
- 解决: 手动编辑冲突文件 → `git add` → `git commit`

**E11: `Your branch is ahead by N commits`**
- 解决: `git pull --rebase` 或 `git push`

**E12: `fatal: not a git repository`**
- 解决: `git init` (在项目根)

### 8.4 数据库常见错误

**E13: `Access denied for user 'root'@'localhost'`**
- 解决: 检查密码
  ```sql
  ALTER USER 'root'@'localhost' IDENTIFIED BY 'new_password';
  ```

**E14: `Unknown database 'minimax_platform'`**
- 解决: 先 `CREATE DATABASE minimax_platform;`

**E15: `Table doesn't exist`**
- 解决: 导入 `sql/init.sql`

## 第 9 章 提交规范

### 9.1 Conventional Commits

格式:
```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type**:
- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档
- `style`: 格式
- `refactor`: 重构
- `perf`: 性能
- `test`: 测试
- `chore`: 杂项
- `build`: 构建

**Scope** (可选, 模块名):
- `ai`, `auth`, `chat`, `gateway`, `frontend` ...

**Subject**: 简短描述, 不超过 50 字, 不加句号

**Body** (可选): 详细说明

**Footer** (可选): 关联 issue

### 9.2 示例

```
feat(ai): V3.0.3 智能体群 (Agent Group) 框架

- 4 角色 (MANAGER/WORKER/CRITIC/OBSERVER) + 4 策略 (PIPELINE/DEBATE/VOTE/SWARM)
- GroupSharedMemory 黑板模式 + GroupMessageBus 消息总线
- GroupOrchestrator 协调引擎
- 7 REST API + 17 单元测试全过

Refs: #123
```

### 9.3 推送前检查清单

- [ ] 代码格式化 (`npm run lint`)
- [ ] 单元测试全过 (`mvn test` / `npm run test:unit`)
- [ ] 无 `console.log` 残留
- [ ] 无 `System.out.println` 残留
- [ ] 无大文件 (≥ 1MB) 提交
- [ ] 无 IDE 配置文件 (`.idea/`, `.vscode/`)
- [ ] Commit message 符合规范
- [ ] 分支名符合规范 (`feature/xxx`)

### 9.4 PR 模板

```markdown
## 改动说明
- 新增: xxx
- 修改: yyy
- 修复: zzz

## 测试
- [ ] 单元测试
- [ ] 手动测试
- [ ] E2E 测试 (若适用)

## 关联 Issue
Closes #123

## 截图 (UI 改动)
![xxx](url)
```

**关键算法 (分块):**
```java
private List<String> splitText(String text, int size, int overlap) {
    String[] paras = text.split("\\n\\n");  // 1. 按段落切
    List<String> chunks = new ArrayList<>();
    StringBuilder cur = new StringBuilder();
    for (String p : paras) {
        if (cur.length() + p.length() + 2 > size) {
            chunks.add(cur.toString());
            // 保留 overlap
            int start = Math.max(0, cur.length() - overlap);
            cur = new StringBuilder(cur.substring(start));
        }
        cur.append(p).append("\n\n");
    }
    if (cur.length() > 0) chunks.add(cur.toString());
    return chunks;
}
```

### 11.2 关键代码: 选举

```java
// 1. 选举超时检查
if (n.getState() != NodeState.LEADER && n.isElectionTimeout()) {
    triggerElection(n);
}

// 2. becomeCandidate
long newTerm = node.becomeCandidate();
node.recordVote();  // 自投
// 3. 广播 RequestVote
for (String peer : node.getPeers()) {
    RaftRpc.RequestVoteResponse resp = nodes.get(peer).handleRequestVote(req);
    if (resp.isVoteGranted()) {
        node.recordVote();
        if (node.hasWonElection()) node.becomeLeader();
    }
}
```

### 11.3 关键代码: 日志复制

```java
// LEADER 提交命令
long newIdx = leader.submitCommand(command);  // 仅 LEADER 成功

// AppendEntries
for (String peer : peers) {
    long prevIdx = leader.getNextIndex(peer) - 1;
    List<LogEntry> entries = leader.getLogEntries().subList(prevIdx+1, ...);
    AppendEntriesResponse resp = nodes.get(peer).handleAppendEntries(req);
    if (resp.success) leader.recordMatchIndex(peer, newIdx);
    else leader.recordMatchFailure(peer);  // nextIndex--
}

// 多数派 commit
long candidate = leader.findCommitIndex();  // 中位数
if (candidate > leader.getCommitIndex().get()) {
    leader.advanceCommit(candidate);
}
```

### 11.4 关键代码: 冲突处理

```java
// Follower 一致性检查
LogEntry prev = getLogAt(req.getPrevLogIndex());
if (prev == null || prev.getTerm() != req.getPrevLogTerm()) {
    return AppendEntriesResponse.builder()
        .success(false)
        .conflictIndex(req.getPrevLogIndex())
        .conflictTerm(prev != null ? prev.getTerm() : 0)
        .build();
}
// LEADER 收到失败 → nextIndex 回退重试
```

### 11.5 常见错误

| 现象 | 原因 | 解决 |
|------|------|------|
| 字段名 log 报错 | 与 Slf4j log 冲突 | 改 `logEntries` |
| 测试 OOM | 选举循环死锁 | 用 try/catch 包裹 RPC |
| 多数派提交没生效 | `findCommitIndex` 没排除旧 term | 加 term 检查 |
| Follower 一直不应用 | 没调用 `applyCommitted` | 提交后立即 apply |

### 12.6 常见错误

|------|------|------|
