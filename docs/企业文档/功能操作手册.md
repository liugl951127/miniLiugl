## 模块 1: AI 智能体

### 1.1 什么是 AI 智能体?

AI 智能体 (Agent) 是一个能自主感知环境、决策、执行任务的程序. 相比普通 AI 工具, 它具备:
- 自主规划能力 (拆解复杂任务)
- 工具调用能力 (ReAct 循环)
- 记忆能力 (上下文+长期)
- 权限管理 (位置感知)

### 1.2 创建智能体

**步骤**:
1. 侧边栏 → "智能体" → "智能体市场"
2. 点击右上角 "创建智能体"
3. 填写基础信息:
   - 名称
   - 描述
   - 头像 (可选)
   - 标签
   - 可见性 (公开/私有/不公开)
4. 配置能力 (Function Tools)
5. 配置提示词
6. 点击"发布"

### 1.3 测试智能体

发布前可在"测试台"对话测试:
1. 智能体详情 → "测试台" tab

## 模块 2: 智能体群组

### 2.1 什么是智能体群?

智能体群 (Agent Group) 是多个 AI 智能体协作完成任务的机制. 4 种协作策略:
- **PIPELINE**: A → B → C 顺序流水线
- **DEBATE**: 多 Agent 辩论, 互相 review, 迭代收敛
- **VOTE**: 多 Agent 独立答案, 多数决
- **SWARM**: 群智, manager 择优

### 2.2 创建群组

**步骤**:
1. 侧边栏 → "智能体群组" → "新建群组"
2. 选择"快速模板"或"自定义"
3. 填写:
   - 群组名
   - 描述
   - 协作策略
   - 成员列表 (角色+权重)
4. 点击"创建"

**预置模板**:
| 模板 | 策略 | 成员 | 适用场景 |
|------|------|------|----------|
| 写作团队 | PIPELINE | 1 manager + 2 writer + 1 reviewer | 文章撰写 |
| 辩论小组 | DEBATE | 1 manager + 3 debater | 方案评估 |
| 投票委员会 | VOTE | 1 manager + 5 voter | 决策 |
| 群智网 | SWARM | 1 manager + 3 worker | 复杂任务 |

### 2.3 执行任务

**步骤**:
1. 进入群组详情
2. 点击"运行任务"
3. 输入主题 + 描述
4. 选择超时时间
5. 点击"开始"
6. 实时查看进度 (WebSocket 推送)
7. 完成后查看结果 + 共识分数

## 模块 4: 模型市场

### 4.1 模型市场功能

- 模型发布: 上传 .onnx / .pt / .h5 模型
- 版本管理: 支持多版本 (1.0 / 1.1 / 2.0)
- License 销售: 4 种 (TRIAL/PERSONAL/COMMERCIAL/ENTERPRISE)
- 计费: 调用次数 / 包月 / 包年
- 评分评价: 5 星 + 文字

### 4.2 发布模型

**步骤**:
1. 模型市场 → "发布模型"
2. 填写模型信息 (名称/描述/分类)
3. 上传模型文件
4. 配置输入/输出 schema
5. 设置 License 类型
6. 提交审核

### 4.3 购买 License

**步骤**:
1. 找到目标模型
2. 点击"购买/订阅"
3. 选择 License 类型
4. 支付 (内部信用 / 支付宝 / 微信)
5. 立即获得 licenseKey

### 4.4 调用模型

```bash
curl -X POST http://localhost:7080/api/v1/ai/market-v2/license/check \
  -H "Content-Type: application/json" \
  -d '{"licenseKey": "lic-xxxxx"}'
```

### 4.5 模型版本管理

- **PUBLISHED**: 当前可用
- **DEPRECATED**: 旧版, 提示升级
- **YANKED**: 紧急下架
- **DRAFT**: 草稿, 未发布

## 模块 5: 训练可视化

### 5.1 功能

- 训练任务管理 (CRUD)
- 实时 loss/accuracy 曲线 (SSE 推送)
- EMA 平滑曲线
- Checkpoint 管理 (保存/下载/最佳)
- 多任务并行

### 5.2 创建训练任务

**REST API**:
```bash
curl -X POST http://localhost:7080/api/v1/ai/training/create \
  -H "Content-Type: application/json" \
  -d '{
    "name": "transformer-base",
    "model": "transformer",
    "totalEpochs": 10,
    "config": "{\"lr\": 0.001}"
  }'
```

### 5.3 上报指标

```bash
curl -X POST http://localhost:7080/api/v1/ai/training/{taskId}/metric \
  -d '{
    "epoch": 1,
    "step": 100,
    "loss": 2.5,
    "valLoss": 2.7,
    "accuracy": 0.65,
    "lr": 0.001,
    "elapsedMs": 60000
  }'
```

### 5.4 查看实时曲线

前端 EventSource 订阅:
```javascript
const sse = new EventSource(`/api/v1/ai/training/${taskId}/stream`)
sse.addEventListener('metric', e => {
  const data = JSON.parse(e.data)
  chart.update(data)
})
```

### 5.5 Checkpoint 操作

```bash
# 保存 (multipart)
curl -X POST .../checkpoint \
  -F "file=@model.bin" \
  -F "name=best-val-loss" \
  -F "epoch=10" \
  -F "step=10000" \
  -F "valLoss=0.25"

# 查最佳
curl .../best

# 下载
curl .../checkpoint/{id}/download -o model.bin
```

## 模块 6: 数据看板

### 6.1 11 个核心指标

| 指标 | 含义 |
|------|------|
| user.total | 总用户数 |
| user.active | 活跃用户 (近 1h) |
| ai.call.count | AI 调用次数 |
| ai.token.total | AI 总 token |
| ai.tool.usage | 工具使用 (按工具分维度) |
| training.running | 运行中训练任务 |
| training.completed | 已完成训练任务 |
| api.requests | API 总请求数 |
| api.errors | API 错误数 |
| cache.hit | 缓存命中 |
| cache.miss | 缓存未命中 |

### 6.2 复合指标

- errorRate = apiErrors / apiRequests
- cacheHitRate = cacheHit / (cacheHit + cacheMiss)
- toolUsage Top N

### 6.3 实时流 (SSE)

```javascript
const sse = new EventSource('/api/v1/ai/dashboard/stream')
sse.addEventListener('snapshot', e => {
  const snapshot = JSON.parse(e.data)
  updateCards(snapshot)
})
```

每 5 秒推送一次.

### 6.4 趋势查询

支持近 N 小时:
- 1h (分钟级)
- 24h (小时级)
- 7d (天级)

## 模块 9: 集群联邦

### 9.1 节点管理

- 自动注册 (启动时)
- 心跳上报 (10s 一次)
- 离线检测 (30s 无心跳)
- 负载监控 (CPU/内存/GPU)

### 9.2 任务路由

4 种路由策略:
- LEAST_LOAD: 选负载最低
- ROUND_ROBIN: 轮询
- CAPABILITY: 按能力匹配
- CURRENT: 本节点

### 9.3 主节点选举

- 自动选举 (最小 ID 优先)
- 5s 重新检测
- 失败自动切换

### 9.4 集群统计

```bash
curl .../api/v1/ai/cluster/stats
```

返回:

  "total": 5,

  "offline": 1,
  "leader": "node-abc123",
  "isCurrentLeader": true,
  "avgCpuUsage": 0.45,

## 模块 10: 移动推送

### 10.1 三平台支持

- **Web Push** (Chrome/Firefox/Edge/Safari)
- **iOS APNs**
- **Android FCM**

### 10.2 注册订阅

Web 端:
```javascript
const reg = await navigator.serviceWorker.ready
const sub = await reg.pushManager.subscribe({
  userVisibleOnly: true,
  applicationServerKey: vapidKey
})
await axios.post('/api/v1/ai/push/subscribe', {
  userId: 1,
  platform: 'web',
  endpoint: sub.endpoint,
  p256dhKey: btoa(String.fromCharCode(...new Uint8Array(sub.getKey('p256dh')))),
  authKey: btoa(String.fromCharCode(...new Uint8Array(sub.getKey('auth'))))
})
```

### 10.3 发送推送

```bash
# 用户
curl -X POST .../api/v1/ai/push/send/user \
  -d '{"userId": 1, "title": "新消息", "body": "您有 3 条未读"}'

# 平台广播
curl -X POST .../api/v1/ai/push/send/platform \
  -d '{"platform": "web", "title": "系统通知", "body": "..."}'

# 全部
curl -X POST .../api/v1/ai/push/send/broadcast \
  -d '{"title": "全员通知", "body": "..."}'
```

### 10.4 限速

- 5 分钟最多 5 条 (同一目标)
- 超出返回 429

## 模块 11: 监控告警

### 11.1 监控维度

- JVM (堆/线程/GC)
- 数据库 (连接池/慢查询)
- 磁盘 (使用率/IO)
- CPU / 内存
- API 响应时间
- 错误率

### 11.2 告警规则

**预置规则**:
- CPU > 90% (critical)
- 内存 > 85% (warning)
- API 错误率 > 5% (warning)
- AI 响应 > 5s (warning)

**自定义规则**:
1. 管理 → 告警规则 → 新建
2. 配置: 指标 / 操作符 / 阈值 / 严重度
3. 启用

### 11.3 告警渠道

- 邮件
- 钉钉
- 飞书
- 企微
- Slack
- Webhook

### 11.4 告警生命周期

```
PENDING → FIRING → ACKNOWLEDGED → RESOLVED
```

## 模块 12: Webhook 集成

### 12.1 创建 Webhook

1. 集成 → Webhook → 新建
2. 填写:
   - 名称
   - URL
   - 事件类型 (agent.created / task.completed / alert.firing ...)
   - 签名密钥
3. 测试连接
4. 启用

### 12.2 事件格式

```json
{
  "event": "task.completed",
  "timestamp": "2026-07-13T10:30:00Z",
  "data": {
    "taskId": "train-123",
    "status": "COMPLETED",
    "metrics": {...}
  },
  "deliveryId": "del-uuid"
}
```

### 12.3 HMAC 签名验证

```
X-Hub-Signature: sha256=<HMAC-SHA256(secret, body)>
```

接收方验证:
```javascript
const crypto = require('crypto')
const expected = 'sha256=' + crypto.createHmac('sha256', secret)
                                 .update(body).digest('hex')
if (sig !== expected) return res.status(401).send('Invalid')
```

### 12.4 重试机制

- 失败自动重试 3 次
- 间隔: 0s / 1s / 4s / 16s (指数退避)
- 仍失败: 标 FAILED, 手动重投
