## 模块 1: AI 智能体

### 1.1 什么是 AI 智能体?

AI 智能体 (Agent) 是一个能自主感知环境、决策、执行任务的程序. 相比普通 AI 工具, 它具备:
- 自主规划能力 (拆解复杂任务)
- 工具调用能力 (ReAct 循环)
- 记忆能力 (上下文+长期)
- 权限管理 (位置感知)

### 1.2 创建智能体

**步骤**:
1. 侧边栏 → "智能体" → "智能体市场"
2. 点击右上角 "创建智能体"
3. 填写基础信息:
   - 名称
   - 描述
   - 头像 (可选)
   - 标签
   - 可见性 (公开/私有/不公开)
4. 配置能力 (Function Tools)
5. 配置提示词
6. 点击"发布"

### 1.3 测试智能体

发布前可在"测试台"对话测试:

## 模块 2: 智能体群组

### 2.1 什么是智能体群?

智能体群 (Agent Group) 是多个 AI 智能体协作完成任务的机制. 4 种协作策略:
- **PIPELINE**: A → B → C 顺序流水线
- **DEBATE**: 多 Agent 辩论, 互相 review, 迭代收敛
- **VOTE**: 多 Agent 独立答案, 多数决
- **SWARM**: 群智, manager 择优

### 2.2 创建群组

**步骤**:
1. 侧边栏 → "智能体群组" → "新建群组"
2. 选择"快速模板"或"自定义"
3. 填写:
   - 群组名
   - 描述
   - 协作策略
   - 成员列表 (角色+权重)
4. 点击"创建"

**预置模板**:
| 模板 | 策略 | 成员 | 适用场景 |
|------|------|------|----------|
| 写作团队 | PIPELINE | 1 manager + 2 writer + 1 reviewer | 文章撰写 |
| 辩论小组 | DEBATE | 1 manager + 3 debater | 方案评估 |
| 投票委员会 | VOTE | 1 manager + 5 voter | 决策 |
| 群智网 | SWARM | 1 manager + 3 worker | 复杂任务 |

### 2.3 执行任务

**步骤**:
1. 进入群组详情
2. 点击"运行任务"
3. 输入主题 + 描述
4. 选择超时时间
5. 点击"开始"
6. 实时查看进度 (WebSocket 推送)
7. 完成后查看结果 + 共识分数

| api.errors | API 错误数 |
| cache.hit | 缓存命中 |
| cache.miss | 缓存未命中 |

### 6.2 复合指标

- errorRate = apiErrors / apiRequests
- cacheHitRate = cacheHit / (cacheHit + cacheMiss)
- toolUsage Top N

### 6.3 实时流 (SSE)

```javascript
const sse = new EventSource('/api/v1/ai/dashboard/stream')
sse.addEventListener('snapshot', e => {
  const snapshot = JSON.parse(e.data)
  updateCards(snapshot)
})
```

每 5 秒推送一次.

### 6.4 趋势查询

支持近 N 小时:
- 1h (分钟级)
- 24h (小时级)
- 7d (天级)

### 12.3 HMAC 签名验证

```
X-Hub-Signature: sha256=<HMAC-SHA256(secret, body)>
```

接收方验证:
```javascript
const crypto = require('crypto')
const expected = 'sha256=' + crypto.createHmac('sha256', secret)
                                 .update(body).digest('hex')
if (sig !== expected) return res.status(401).send('Invalid')
```

### 12.4 重试机制

- 失败自动重试 3 次
- 间隔: 0s / 1s / 4s / 16s (指数退避)
- 仍失败: 标 FAILED, 手动重投

## 模块 13: V3.4.0 自研知识库

### 13.1 功能描述

MiniMax 自研知识库是一个企业级 RAG 系统, 提供:
- 文档上传 (SHA256 去重)
- 智能分块 (按段落 + 重叠, 默认 500/50)
- 关键词 + 模拟向量化 (384 维, 实际生产对接 EmbeddingModel)
- 混合检索 (TF 关键词 + 余弦相似度 + RRF 融合, k=60)
- 引用追踪 (chunkId/docId/filename/seq)
- RBAC 权限 (READ/WRITE/ADMIN)

### 13.2 数据模型

3 张表:
- `kb_document` - 文档元信息 (docId/kbId/filename/sha256/status/chunkCount/embeddingCount)
- `kb_chunk` - 文档分块 (seq/content/embedding JSON/embeddingModel/keywords/summary)
- `kb_permission` - 权限 (subjectType: USER/ROLE/ORG, permission)

文档状态机: `PENDING → PARSING → INDEXED / FAILED / DELETED`

### 13.3 REST API (11 个)

| Method | URL | 说明 |
|--------|-----|------|
| POST | `/api/v1/ai/kb/upload` | 上传文档 (multipart) |
| GET | `/api/v1/ai/kb/docs/{docId}` | 查文档 |
| GET | `/api/v1/ai/kb/docs/list/{kbId}` | 列知识库文档 |
| DELETE | `/api/v1/ai/kb/docs/{docId}` | 删文档 |
| GET | `/api/v1/ai/kb/chunks/{docId}` | 列分块 |
| POST | `/api/v1/ai/kb/search` | 混合检索 (RRF) |
| POST | `/api/v1/ai/kb/search/keyword` | 纯关键词检索 |
| GET | `/api/v1/ai/kb/stats/{kbId}` | 知识库统计 |
| POST | `/api/v1/ai/kb/permission/grant` | 授权 |
| POST | `/api/v1/ai/kb/permission/revoke` | 撤销 |
| GET | `/api/v1/ai/kb/public` | 公开文档列表 |

### 13.4 算法详解

**分块 (splitText)**:
1. 按段落 `\\n\\n` 切分
2. 段落 < 500 字符, 合并到当前块
3. 段落 > 500 字符, 硬切 (保留 overlap 50)
4. 最后一个块, 拼接后段尾部 (overlap)

**向量化 (mockEmbedding)**:
1. 字符 hash → 384 维 float
2. L2 归一化 (除以 sqrt(sum(x²)))

**混合检索 (hybridSearch)**:
1. 关键词检索: TF 评分, `score = ∑(hit+1)`
2. 向量检索: 余弦相似度
3. RRF 融合: `final_score = ∑ 1/(k + rank)` (k=60)
4. 按 final_score 降序, 取 topK

## 模块 16: V3.5.0 Raft 分布式一致性

### 16.1 功能描述

`RaftNode` + `RaftCluster` 实现标准 Raft 算法 (Diego Ongaro 2014), 用于集群联邦的一致性协调.

### 16.2 核心数据结构

- `currentTerm` (AtomicLong): 当前任期
- `votedFor`: 本任期投票对象
- `logEntries`: 日志列表 (index 0 是占位)
- `commitIndex` / `lastApplied`: 提交 / 应用进度
- `nextIndex` / `matchIndex` (LEADER): 每个 Follower 复制状态

### 16.3 3 种节点状态

| 状态 | 角色 | 行为 |
|------|------|------|
| FOLLOWER | 跟随者 | 默认状态, 接收心跳 |
| CANDIDATE | 候选者 | 选举超时, 发起拉票 |
| LEADER | 领导者 | 处理写, 复制日志 |

### 16.4 4 类 RPC

| RPC | 方向 | 用途 |
|-----|------|------|
| RequestVote | CANDIDATE → 全部 | 拉票 |
| AppendEntries | LEADER → 全部 | 心跳 / 日志复制 |
| InstallSnapshot | LEADER → 落后节点 | 发送快照 |
| (响应) | 反向 | 投票 / 追加结果 |

### 16.5 关键算法

**选举规则 (§5.4):**
1. RPC term < currentTerm → 拒绝
2. votedFor == null || votedFor == candidateId → 可投
3. 候选人日志至少一样新 → 可投

**日志复制:**
1. LEADER 收到命令 → 加入本地 log
2. 发送 AppendEntries 到所有 Follower
3. 多数派成功后 → commit + apply
4. 下一轮心跳 / 复制

**冲突处理:**
- Follower 收到 prevLogIndex term 不匹配 → 返回 conflictIndex
- LEADER 把 nextIndex 回退, 重试

## 模块 17: V3.5.1 推送真实集成

### 17.1 功能描述

`PushIntegrationService` 协调 3 个真实 Provider:
- `WebPushProvider` (RFC 8030 + VAPID ES256)
- `ApnsProvider` (HTTP/2 + JWT ES256)
- `FcmProvider` (HTTP v1 + OAuth2)

### 17.2 VAPID 协议

**密钥对 (P-256 ECDSA):**
- 公钥: 65 字节 uncompressed point → 88 字符 base64url
- 私钥: 32 字节 → 43 字符 base64url

**JWT 结构:**
```
header:  {"typ":"JWT","alg":"ES256"}
payload: {"aud":"<endpoint>","exp":<ts>,"sub":"mailto:..."}
sig:     ES256(header.payload, privKey)
```

**签名转换:** DER → JOSE 紧凑格式 (R || S, 各 32 字节)

### 17.3 APNs JWT (ES256, 1h 缓存)

```java
header:  {"alg":"ES256","kid":"<keyId>"}
payload: {"iss":"<teamId>","iat":<ts>}
sig:     ES256(header.payload, privKey)
```

### 17.4 FCM HTTP v1 协议

**OAuth2 流程:**
1. Service Account JSON 解析私钥
2. 用私钥签 RS256 JWT (iss=client_email, scope=firebase.messaging)
3. POST https://oauth2.googleapis.com/token 换 access_token
4. Bearer access_token 调 FCM v1 API

**请求体:**
```json
{
  "message": {
    "token": "...",
    "notification": {"title": "...", "body": "..."},
    "android": {"priority": "HIGH", "ttl": "60s"},
    "webpush": {"headers": {"Urgency": "high"}},
    "data": {"key": "value"}
  }
}
```
