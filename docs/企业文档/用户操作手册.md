## 第 6 章 常见问题 FAQ

### Q1: 忘记密码怎么办?

**答**:
1. 登录页点击"忘记密码"
2. 输入注册邮箱
3. 查收重置邮件
4. 点击链接重置 (1 小时内有效)
5. 如未收到, 联系管理员重置

### Q2: AI 响应很慢?

**答**:
1. 检查网络连接
2. 切换到"内置"模型
3. 避开高峰时段
4. 联系管理员检查 GPU 资源

### Q3: 上传文件失败?

**答**:
1. 检查文件大小 (默认 ≤ 100MB)
2. 检查文件格式 (PDF/Word/Excel/图片)
3. 清除浏览器缓存重试
4. 切换浏览器 (Chrome 80+)

### Q4: 工具调用报错?

**答**:
1. 检查输入参数
2. 查看错误码 (工具详情页有说明)
3. 联系技术支持 (附上 traceId)

### Q5: 如何邀请同事?

**答**:
1. 管理员登录
2. 管理 → 用户管理 → 新建用户
3. 填写信息, 分配角色
4. 系统自动发送激活邮件

### Q6: 支持哪些浏览器?

**答**: Chrome 80+ / Edge 79+ / Firefox 70+ / Safari 13+ (推荐 Chrome)

### Q7: 支持移动端吗?

**答**: 支持
- 浏览器访问: 自动适配
- 原生 App: 见 [移动端开发文档](../MOBILE.md) (iOS/Android via Capacitor)

### Q8: 数据是否安全?

**答**:
- 数据本地存储 (企业内网)
- 传输加密 (HTTPS / TLS 1.2+)
- 密码 BCrypt 哈希
- 审计日志全留痕
- 详见 [安全白皮书](#) (企业版)

## 第 7 章 🆕 V3.4 自研知识库

### 7.1 什么是自研知识库?

MiniMax 平台 V3.4.0 上线的 **自研知识库 (Knowledge Base)** 是一个完整的 RAG 系统, 让企业可以:

- 上传产品文档 / 操作手册 / 业务资料
- 自动分块、向量化、构建索引
- 客户提问时, 自动检索最相关的文档片段, 引用回答

### 7.2 上传文档

1. 登录 → 顶部菜单 **AI 平台** → **自研知识库**
2. 选择 **知识库** (如 "产品手册 KB")
3. 点击 **上传文档**, 选择本地文件 (支持 txt/pdf/md/docx)
4. 等待状态变为 `INDEXED` (自动分块+向量化完成)

### 7.3 知识库检索

1. 进入知识库, 在 **搜索框** 输入客户问题
2. 系统返回 top-5 相关的文档片段, 含 **来源/页码/引用**
3. 点击片段可查看完整文档

### 7.4 权限管理

知识库支持 3 类权限: READ / WRITE / ADMIN, 可授予用户/角色/部门.

```
POST /api/v1/ai/kb/permission/grant
{
  "kbId": "product-kb",
  "subjectType": "USER",
  "subjectId": 1001,
  "permission": "READ"
}
```

## 第 8 章 🆕 V3.4 客户意图精准预测

### 8.1 什么是意图预测?

客户输入一句话 ("我要退款, 紧急!"), 平台会立即:

| 维度 | 输出示例 |
|------|---------|
| **意图** | `complaint` (投诉) |
| **置信度** | 0.85 |
| **情感** | `negative` (负面) |
| **紧迫度** | 0.6 (高) |
| **实体** | `[time: 明天, money: 100元, phone: 138...]` |
| **推荐 Agent** | `echo-reviewer` (客服审查员) |

### 8.2 使用方式

#### 方式 1: 实时 API

```bash
curl -X POST http://localhost/api/v1/ai/intent/predict \
  -H "Content-Type: application/json" \
  -d '{"text":"我要退款, 紧急!"}'
```

#### 方式 2: AI 对话自动启用

在 AI 聊天窗口中, 每条消息自动经过意图预测, 头部显示:
- 🎯 意图标签
- ⚠️ 紧迫度 (高/中/低)
- 😊 情感 (正面/负面/中性)

### 8.3 8 大意图分类

| 意图 | 典型关键词 | 推荐 Agent |
|------|----------|------------|
| query (查询) | 查询/多少/几个 | echo-analyzer |
| order (下单) | 买/下单/购买 | echo-writer |
| complaint (投诉) | 投诉/退款/差评 | echo-reviewer |
| consult (咨询) | 咨询/请问/怎么 | echo-translator |
| cancel (取消) | 取消/撤销 | echo-writer |
| feedback (反馈) | 反馈/建议 | echo-summarizer |
| pay (支付) | 付款/支付 | echo-reviewer |
| login (登录) | 登录/注册 | echo-coder |

### 8.4 自定义关键词

```bash
POST /api/v1/ai/intent/keyword
{"intent":"query","keyword":"黄金","weight":100.0}
```

## 第 9 章 🆕 V3.4 一句话生成智能体群

### 9.1 什么是智能体群自动生成?

传统 AI 平台需要手动选 Agent 配参数, MiniMax V3.4.2 让用户**一句话**生成完整群组:

> 输入: "写一份季报"
> 自动生成: 写作团队 (1 经理 + 2 写作员 + 1 润色员, PIPELINE 策略)

### 9.2 使用方式

1. 进入 **AI 平台** → **智能体群**
2. 点击 **🪄 一句话生成**
3. 输入任务描述: "分析用户行为" / "用 Java 写代码" / "对比两种方案"
4. 平台自动:
   - 识别意图 (分析/编码/辩论/...)
   - 选择模板 (6 个内置模板之一)
   - 拼装 Agent 群 (MANAGER + WORKER + CRITIC)
   - 选择协作策略 (PIPELINE/VOTE/DEBATE/SWARM)
5. 立即执行, 输出结果

### 9.3 6 大内置模板

| 模板 | 策略 | Agent 组成 | 适用场景 |
|------|------|----------|---------|
| WRITING_TEAM | PIPELINE | 1 经理 + 2 写作员 + 1 润色员 | 写文档/报告/总结 |
| ANALYST_TEAM | SWARM | 1 经理 + 3 分析员 (数据/市场/风险) | 多维分析 |
| DEBATE_PANEL | DEBATE | 1 主持 + 2 辩手 + 1 评委 | 对比/辩论 |
| VOTE_COUNCIL | VOTE | 1 主席 + 3 投票者 | 决策/投票 |
| CODER_TEAM | PIPELINE | 1 技术负责人 + 1 架构师 + 1 程序员 + 1 评审 | 编码 |
| RESEARCH_TEAM | SWARM | 1 研究负责人 + 3 探索员 | 研究/调研 |

### 9.4 API 用法

```bash
# 1. 一句话生成
POST /api/v1/ai/agent-group/auto/generate
{"oneLiner":"写一份季报"}

# 2. 按模板生成
POST /api/v1/ai/agent-group/auto/template
{"template":"WRITING_TEAM","description":"写用户使用手册"}

# 3. 列出所有模板
GET /api/v1/ai/agent-group/auto/templates
```

### 9.5 4 大协作策略

| 策略 | 流程 | 适用 |
|------|------|------|
| **PIPELINE** | A → B → C 顺序执行 | 流水线/编码 |
| **VOTE** | 多个 Agent 并行, 投票 | 决策/评估 |
| **DEBATE** | 正反方辩论, 评审判定 | 对比/分析 |
| **SWARM** | 多 Agent 自由贡献, Manager 择优 | 多维分析 |

## 第 10 章 🆕 V3.5 Raft 分布式一致性

### 10.1 什么是 Raft?

Raft 是一种分布式一致性算法, 让多个节点对一份数据达成共识. MiniMax 平台用 Raft 实现:
- 集群 Leader 选举 (主节点自动选出)
- 写请求自动复制到多数派才提交
- 节点掉线 / 恢复自动重选
- 数据强一致 (commit 后不会丢)

### 10.2 3 节点集群

平台默认启动 3 节点 Raft 集群 (node-1, node-2, node-3):

1. 进入 **AI 平台** → **集群联邦** → **Raft 集群**
2. 点击 **启动集群**, 等待 2 秒

## 第 11 章 🆕 V3.5.1 推送真实集成

### 11.1 三平台真实协议

V3.5.1 起, MiniMax 平台对接三大推送平台真实协议, **不再 mock**:

| 平台 | 协议 | 鉴权 |
|------|------|------|
| Web Push | RFC 8030 + RFC 8292 + RFC 8291 | VAPID (ES256 P-256) |
| APNs | HTTP/2 + JSON | JWT (ES256, keyId/teamId) |
| FCM | HTTP v1 API | OAuth 2.0 (Service Account) |

### 11.2 自动平台检测

```bash
# 智能识别 target 类型, 自动选 Provider
POST /api/v1/ai/push/integration/auto
{
  "target": "https://fcm.googleapis.com/wp/abc123",
  "title": "新订单",
  "body": "您有 1 笔新订单",
  "data": {"orderId": "12345"}
}
```

平台识别规则:
- `https://...` → Web Push
- 64 位 hex → APNs
- 其他 → FCM

### 11.3 三平台单独调用

```bash
# Web Push
POST /api/v1/ai/push/integration/web
{ "target": "https://...", "title": "...", "body": "..." }

# APNs
POST /api/v1/ai/push/integration/apns
{ "target": "abcdef0123...", "title": "...", "body": "..." }

# FCM
POST /api/v1/ai/push/integration/fcm
{ "target": "fcm-token", "title": "...", "body": "..." }
```

### 11.4 VAPID 集成

```bash
# 1. 获取 VAPID 公钥 (前端 subscribe 用)
GET /api/v1/ai/push/integration/vapid-public-key

# 2. 前端用公钥 subscribe
navigator.serviceWorker.register(...).then(reg =>
  reg.pushManager.subscribe({
    userVisibleOnly: true,
    applicationServerKey: urlBase64ToUint8Array(publicKey)
  })
)

# 3. 把 endpoint 传给后端, 后端用私钥签 VAPID JWT 推送
```
